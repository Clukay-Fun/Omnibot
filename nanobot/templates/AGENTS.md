# AGENTS.md - 你的工作区

这个工作区承载你的连续性。请按真实文件职责使用它，不要自行发明新的默认分层。

## 文件职责

- `WORKLOG.md`
  记录有明确下一步、可推进、可完成的事项。当前工作状态变化时，优先更新这里。
- `USER.md`
  记录稳定偏好、沟通方式、协作习惯、风险与确认偏好。
- `memory/MEMORY.md`
  记录长期有效的事实、长期工作背景、长期关系信息。
- `memory/HISTORY.md`
  只用于回查和追溯过去事件，不是普通对话的默认上下文。
- `HEARTBEAT.md`
  记录低打扰提醒规则与周期性检查约定，不承载完整任务正文。

## 极简判断规则

- 有明确下一步、会完成的事项 -> `WORKLOG.md`
- 稳定偏好、长期背景、长期有效事实 -> `USER.md` / `memory/MEMORY.md`
- 仅用于回查的事件摘要 -> `memory/HISTORY.md`

## 工作方式

- 当系统 prompt 给出了 user-scoped 路径时，始终以那个路径为准。
- 不要假设存在 `memory/YYYY-MM-DD.md`、今日笔记、昨日笔记或会话启动必读日记流。
- 普通对话先回复用户；如果本轮改变了工作状态或长期事实，再在同一轮维护对应文件。
- 不要把聊天记录本身当作稳定记忆；需要保留的内容应写入对应文件。

## 红线

- 不要泄露隐私数据。
- 不要在未询问的情况下执行破坏性命令。
- `trash` > `rm`
- 有疑问时先确认，不要擅自做高风险外部操作。
