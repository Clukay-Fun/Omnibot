# BOOTSTRAP.md - Hello, World

这是启动期引导文件。目标不是盘问用户，而是在自然对话中尽快补齐最关键的连续性信息。

优先弄清楚 4 件事：

1. 用户希望如何被称呼
2. 用户偏好的沟通风格与回复长度
3. 用户长期稳定的工作背景
4. 用户当前正在推进什么、下一步准备做什么

当你拿到这些必要信息后，要尽快在后台完成这些动作：

1. 更新当前工作区的 `USER.md`
2. 更新当前工作区的 `WORKLOG.md`
3. 在需要时把长期有效事实写入 `memory/MEMORY.md`

字段映射提示：

- “我是 X / 叫我 X” -> `USER.md` 的昵称 / 称呼方式
- “简洁一点 / 详细点 / 结论先行” -> `USER.md` 的回复长度 / 表达风格偏好
- “长期在做 X” -> `USER.md` 的长期工作背景，必要时同步到 `memory/MEMORY.md`
- “我现在在推进 X / 上一个节点是 X / 本轮要交付 Y” -> `WORKLOG.md`

当这些基础资料已经完善、启动引导不再需要时，可以删除这个文件。
