# Weekly Report Workflow

这是 `feishu-workspace` 下的受控工作流能力：根据一张飞书多维表格生成本周周报文档，并给当前飞书发起人加 `edit` 权限。

## 何时使用

- 用户明确要从飞书多维表格生成周报。
- 用户给了 Bitable 链接，或给了可解析的 `app_token / table_id`。
- 用户希望最终产出飞书文档，而不是只要一段聊天摘要。

## 输入契约

- v1 每次只处理一张表。
- 默认统计当前自然周，按当前会话时区取值。
- 用户显式给开始/结束日期时，覆盖默认周范围。
- 资源标识不足时先追问，不要猜。

## 工作流程

1. 解析并锁定一张 Bitable。
2. 读取字段列表、字段类型和样例记录。
3. 识别关键字段映射，至少确认日期字段与标题/事项字段。
4. 多个日期字段时，优先选择语义更接近“更新 / 修改 / 完成”的字段，而不是“创建时间”。
5. 字段名是拼音、缩写或语义不清时，最多追问 2 轮；第 3 次必须转成明确字段确认问题。
6. 使用 `feishu-workspace` 完成：
   - `bitable` 读取
   - `doc create`
   - `doc create_blocks`
   - `permission member create`

## 中断与续接

恢复时先简短重述：

- 已确认的字段映射
- 当前统计时间范围
- 仍待确认的问题

不要重新从头盘问整张表。

## 输出要求

最终回复必须包含：

- 文档标题
- 文档链接
- 统计时间范围
- 识别到的关键字段映射
- 是否已完成授权

如果失败，要明确指出失败点，不要编造周报。
