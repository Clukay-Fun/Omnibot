# 更新日志

此文件用于记录项目中所有值得关注的变更。

## 0.3.3 - 2026-03-23

### 近期新增

- 调整 Feishu 私聊用户工作区的初始化判定，只有在用户档案、长期背景和当前工作都补齐后才结束 `BOOTSTRAP.md` 引导。
- 统一长期记忆相关模板与记忆整理提示的中文表达，进一步明确 `MEMORY.md` 与 `WORKLOG.md` 的职责边界。
- 优化 Feishu 私聊模板默认文案和工作区说明，使默认文件命名、标题与界面表述更一致。

## 0.3.2 - 2026-03-22

### 近期新增

- 优化 Feishu 私聊中的公开进度提示规则，要求仅在多步工具操作前发送简短动作说明，避免使用“正在处理”“稍后给你结果”这类占位话术作为正式回复。
- 调整 Feishu 流式回复策略，移除框架级延迟文本提示，统一回到 thinking card 驱动的进度展示。
- 更新 `reset --user-id` 行为，删除目标用户的 Feishu 私聊状态时不再自动重建其 overlay 工作区，避免误留下空白状态目录。

## 0.3.1 - 2026-03-21

### 近期新增

- 将系统上下文进一步收敛为 `规则层 > WORKLOG > MEMORY` 的分层结构，降低 prompt 冲突与记忆污染。
- 收紧 heartbeat 执行边界，限制其仅做读取与提醒发送，不再允许改写任务、人格或长期记忆文件。
- 补充模板职责说明，明确 `WORKLOG.md`、`USER.md`、`memory/MEMORY.md`、`memory/HISTORY.md` 与 `HEARTBEAT.md` 的使用边界。
- 强化“记住”语义，要求在同一轮将稳定偏好或长期背景真正写入对应文件，而不是只停留在回复里。

## 0.3.0 - 2026-03-20

这次升级正式把 Omnibot 推进到“可持续跟踪工作状态”的新阶段。

### 里程碑说明

- 新增 `WORKLOG.md` 工作面板，作为当前任务、优先级与下一步的统一事实源。
- 将 worklog 接入系统 prompt、heartbeat、doctor 检查、Feishu 用户工作区与文件工具，形成完整闭环。
- 明确长期记忆与当前任务的边界，避免把临时执行事项错误沉淀进 `MEMORY.md`。
- 强化私聊场景下的 per-user 上下文隔离，减少多来源摘要混入造成的噪音。

## 0.2.0 - 2026-03-19

### 新增

- 新增完整的 Feishu 集成能力，包括消息接入、媒体处理、路由、记忆、流式回复、交互式思维卡片与广播能力。
- 新增 Feishu Workspace 技能，支持 Calendar、Bitable、Docs 等工作区操作。
- 新增 Feishu OCR 技能，支持图片内容识别相关工作流。
- 新增 `http-api` 技能，支持在技能中发起带认证的 HTTP 调用，并补充更灵活的 Feishu API 访问辅助能力。
- 新增 Feishu 主动通知能力，并支持使用 Card 2.0 渲染外发消息。
- 新增按用户维度的 overlay context、workspace 与引导模板能力，用于个性化记忆与初始化。
- 新增 heartbeat 相关 CLI 控制命令，以及用于环境和配置检查的 `doctor` 命令。
- 新增一键服务器更新脚本，并补充 Feishu 部署、配置、巡检与运维文档。

### 变更

- 将包版本统一收敛为 `nanobot/version.py` 中的单一来源。
- 更新 `nanobot --version` 输出，在可用时附带短 Git 提交号，例如 `nanobot v0.2.0 (f7ab86f)`。
- 明确后续面向发布的版本说明统一记录在本变更日志中。
- 重写并聚焦 README 与部署文档，突出 Feishu 专属智能体与服务器部署路径。
- 更新默认模板与初始化引导内容，补充 `BOOTSTRAP.md` 等预设模板。
- 重构 heartbeat 执行方式，改为在全新的临时会话中运行，并压缩历史会话记录。
- 优化 Feishu 回复卡片、进度反馈与思维卡片展示体验。
- 清理仓库范围内的 Ruff 问题，并移除已被跟踪的 `.DS_Store` 文件。

### 修复

- 加强对 `~` 展开后工作区路径的执行保护。
- 在 MCP 连接路径中捕获 `BaseException`，避免连接取消时导致 agent loop 崩溃。
- 增强异常处理，避免 agent loop 因部分运行时异常直接退出。
